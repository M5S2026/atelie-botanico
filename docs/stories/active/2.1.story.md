# Story 2.1: Supabase Auth — Email + Google OAuth

**Epic:** 2 — Auth & Pagamentos
**Status:** Draft
**Prioridade:** Alta

## Contexto

Todos os planos (Florzinha/Artista/Pro) precisam de autenticação para controlar acesso a features. Supabase free tier suporta até 50k usuários.

## Acceptance Criteria

- [ ] Supabase project criado e configurado
- [ ] Login/signup com email + senha
- [ ] Login com Google OAuth
- [ ] Tela de login/signup com design consistente (verde escuro + verde menta)
- [ ] Proteção de rotas: sem auth → redireciona para login
- [ ] Plano default "florzinha" ao criar conta
- [ ] Perfil básico: nome, email, plano ativo
- [ ] Logout funcional
- [ ] Sessão persistente (refresh token)

## Technical Notes

- `@supabase/supabase-js` como dependência
- Auth context provider em `src/lib/supabase.js`
- RLS policies para isolar dados por usuário
- `.env` com SUPABASE_URL e SUPABASE_ANON_KEY
