# Story 2.2: Dual Payment — Stripe + PayPal Checkout

**Epic:** 2 — Auth & Pagamentos
**Status:** Draft
**Prioridade:** Alta

## Contexto

App precisa de monetização com 2 planos pagos. Dual payment para cobrir Brasil (Stripe com Pix/boleto/cartão + PayPal) e Japão (Stripe com JPY + PayPal). Usuária escolhe o método na hora do checkout.

## Acceptance Criteria

- [ ] Stripe products criados: Artista (subscription BRL + JPY) e Pro (one-time BRL + JPY)
- [ ] PayPal products criados: Artista (subscription) e Pro (one-time)
- [ ] Página de planos com os 3 cards (Florzinha/Artista/Pro)
- [ ] Seletor de método de pagamento: Stripe ou PayPal
- [ ] Stripe Checkout: cartão, Pix, boleto (BR) / cartão (JP)
- [ ] PayPal Checkout: cartão, saldo PayPal
- [ ] Detecção automática de moeda por locale (BRL default, JPY se ja_JP)
- [ ] Webhook Stripe → atualiza plano no Supabase
- [ ] Webhook PayPal → atualiza plano no Supabase
- [ ] Feature gating: componentes verificam plano antes de liberar
- [ ] Portal do cliente para gerenciar assinatura (ambos providers)
- [ ] Tratamento de falha/cancelamento (downgrade para Florzinha)

## Technical Notes

- Stripe Checkout (hosted) — sem formulário custom
- PayPal Buttons SDK — embedded no app
- Supabase tabela `subscriptions` com campo `provider` (stripe/paypal)
- Edge Functions para webhooks de ambos
- `usePlan()` hook agnóstico ao provider
